webpack测试

Jun 2nd 2023

webpack使用：
案例说明：
编写ES6 Moudle代码，
由于浏览器并不认识，就无法解析，
我们可以使用webpack来编译使浏览器认识。

一、环境准备工作：
1.输入命令node -v 查看node的版本号
2.输入命令npm -v 查看npm的版本号
3.输入命令：webpack -v 查看webpack的版本号
以上三个指令均可正常运行再执行下面操作

二、项目准备工作：
1.在d盘/桌面新建文件夹：vuecildemo，存放项目代码

2.在vscode打开刚才新建的文件夹，
在文件夹内部新建文件夹：day01webpack

3.在day01webpack文件夹下分别新建：
1.src目录：存放项目主要js代码
2.dist目录：存放打包后的代码
3.主运行文件：index.html

三、项目代码编写：
1.在src下新建两个js：test.js、index.js

2.在test.js中编写hello两个方法,
function hello(){
console.log(“hello,world…”)
}

使用module导出：
module.exports = {hello};

3.在index.js中导入test.js
const {hello} = require(“./test”);
并调用：
hello();

4.在主文件：index.html中引用index.js

5.尝试运行：运行失败,错误：require is not defined

四、使用webpack将项目进行打包
1.选中项目，右键open in integrated terminal,打开控制台

控制台路径：
C:\Users\Hi\Desktop\day01webpack>
运行：get-ExecutionPolicy
get-ExecutionPolicy（获取当前会话中的执行策略）

如果是:Restricted（有限的），则表示禁止，
则需以管理员的方式打开vscode运行该指令
再运行：set-ExecutionPolicy RemoteSigned；

如果是：RemoteSigned(远程签名)
则继续向下执行

2.运行：npm init
初始化后的文件夹,里面只有一个json文件

以下信息可直接回车跳过
package name: (day01webpack)
version: (1.0.0)
description:
entry point: (index.js)
test command:
git repository:
keywords:
author:
license: (ISC)

3.运行： npm install webpack –save-dev
多了 node_modules 这个文件

4.全局安装：npm install –save-dev webpack-cli -g

5.运行：npm install -D webpack-cli

6.修改：
npm init 创建的package.json文件，
在scripts中添加：
“dev”: “webpack –mode deveplopment”,
“build”: “webpack –mode production”

五、测试
1.运行（需要切换到src目录）：
npx webpack ./index.js -o ../dist –mode development

2.修改index.html中js的引用为dist下的index.js

3.运行

注：打包发布后，则以后修改代码都需要重新打包发布
重新执行：
npx webpack ./index.js -o ../dist –mode development

除非运行实时监听的插件

六、总结：
通过以上按钮我们可以简单理解一下webpakc的基本概念(静态模块打包器)就是将webpack中各种模块资源进行打包合并成一个或多个包.
如我们的index.js和text.js的内容就都打包在了dist文件夹下的main.js中

什么场景下用?
如果你正在用许多非代码的静态资源（比如CSS，图像，字体等）构建一个复杂
的前端应用程序，那么Webpack会给你带来很大的好处。

如果你的应用程序非常小，并且你没有什么的静态资源，

并且你只需要一个JS文件就足够客户端需求，那么Webpack就没有必要使用。

webpack的好处：
清晰简明的模块。一个稍微有点规模的应用往往有着一个复杂的资源关系网，
在需要做优化的时候就会使一件非常的头疼的事情。
webpack可以代替你做这个麻烦事，
将复杂的多个文件打包成你希望的代码模块，引入会非常清晰，
想对引入文件做点什么操作也会更容易。
代码拆分来做资源异步加载。
webpack打包成一个模块之后也带来一个问题，如果加载模块过于庞大，
加载的速度就没法保证，所以webpack也提供了拆分代码后异步加载的方式，
即先加载某个主要模块，当用到某个需要资源异步加载的模块时在对其发送请求加载，这样做在大型项目中很常见。
消除对未引用资源的依赖。
在复杂应用又多静态资源的情况下会减少很多无效的浪费，
这会对应用的有很大提升。可以控制资源的处理方式。
